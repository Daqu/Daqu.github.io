# treat IO request as a message

## introduction

业务需求 -> 拦截、重定向、复制IO请求 -> 添加新功能

实现 -> 需要对文件系统进行修改

## 问题

修改文件系统会导致额外的工作量

-> 也会导致io路径变长

## 贡献

把io请求当作消息处理能够显著降低为文件系统添加新功能时需要的工作量。

-> io path不变长

-> 也能增加新功能

- 怎么做
- 提升
- 额外
